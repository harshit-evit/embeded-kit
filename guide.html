Getting Started with STM32CubeIDE

Complete Setup Guide for Blue Pill Development

STM32CubeIDE is an integrated development environment (IDE) for STM32 microcontrollers, combining code editing, compilation, debugging, and peripheral configuration in one tool. This section provides step-by-step instructions for development with the Blue Pill (STM32F103C8T6).

Section 7.1

Installing STM32CubeIDE

Download and install the official STM32 development environment

✓
1
💻 Download STM32CubeIDE

Visit the official STMicroelectronics website and download the appropriate version:

🔗 Download Link

https://www.st.com/en/development-tools/stm32cubeide.html

  • Select the correct version for your OS (Windows/Linux/macOS)
  • Sign in or create an ST account to download
  • Choose the latest stable version
✓
2
⚙️ Install STM32CubeIDE

Run the downloaded installer and complete the installation process:

  • Run the installer with administrator privileges
  • Follow the on-screen installation wizard
  • Choose default settings unless specific modifications needed
  • Allow firewall exceptions if prompted
⚠️ Important Note

Installation may take 10-15 minutes depending on your system. Ensure you have at least 2GB of free disk space.

Section 7.2

Setting Up a New Project

Create and configure your first STM32 project for the Blue Pill

✓
1
🚀 Launch STM32CubeIDE

Open the installed STM32CubeIDE application from your system:

  • Launch STM32CubeIDE from Start Menu or Applications
  • Select or create a workspace directory
  • Wait for the IDE to fully load
✓
2
📁 Create New Project

Create a new STM32 project for the Blue Pill microcontroller:

  • Click File → New → STM32 Project
  • In MCU/Board Selector, search for STM32F103C8T6
  • Select the microcontroller from the list
  • Click Next to proceed
💡 Tip

You can also search for "Blue Pill" in the board selector for quick access.

✓
3
⚙️ Configure Project Settings

Set up your project configuration and generate the initial code:

  • Enter a meaningful Project Name
  • Select C as the programming language
  • Choose project location (or use default)
  • Click Finish to generate the project
Project Name: MySTM32Project Language: C Toolchain: STM32CubeIDE
Section 7.3

Configuring Peripherals in STM32CubeMX

Set up GPIO, UART, I2C, SPI, and other peripheral interfaces

✓
1
🔧 Enable Peripherals

Configure the required peripherals in the CubeMX interface:

  • Navigate to Pinout & Configuration tab
  • Enable GPIO pins for LEDs and buttons
  • Configure UART for serial communication
  • Set up I2C for sensor interfaces
  • Enable SPI if needed
  • Configure ADC for analog readings
🎯 Common Configurations

PA5: GPIO Output (LED), PC13: GPIO Input (Button), PA9/PA10: USART1

✓
2
⚡ Generate Code

Generate the initialization code for your configured peripherals:

  • Click the "Generate Code" button (gear icon)
  • Wait for code generation to complete
  • Review the generated files in Project Explorer
  • Check main.c for initialization functions
/* Generated initialization code will appear in main.c */ int main(void) { HAL_Init(); SystemClock_Config(); MX_GPIO_Init(); MX_USART1_UART_Init(); // Your code here }
Section 7.4

Compiling and Flashing Code

Build your project and upload it to the Blue Pill microcontroller

✓
1
✏️ Write Your Code

Add your custom application logic to the main.c file:

  • Open main.c in the code editor
  • Add your code inside the while(1) loop
  • Use HAL library functions for peripheral control
/* USER CODE BEGIN 3 */ while (1) { HAL_GPIO_TogglePin(GPIOC, GPIO_PIN_13); HAL_Delay(1000); } /* USER CODE END 3 */
✓
2
🔨 Compile the Project

Build your project to check for errors and generate the binary:

  • Click Project → Build Project or press Ctrl + B
  • Monitor the Console tab for compilation messages
  • Ensure no errors appear in the build output
  • Check that binary files are generated
⚠️ Troubleshooting

If compilation fails, check for syntax errors, missing includes, or incorrect pin configurations.

✓
3
📡 Flash Code to Blue Pill

Upload your compiled program to the STM32F103C8T6 microcontroller:

  • Connect the ST-Link/V2 programmer to the Blue Pill
  • Ensure proper wiring: 3.3V, GND, SWDIO, SWCLK
  • Click Run → Run Configurations
  • Select ST-Link Debugging and click Run
🔌 Wiring Reference

ST-Link: 3.3V→3.3V, GND→GND, SWDIO→DIO, SWCLK→CLK

Section 7.5

Debugging in STM32CubeIDE

Use powerful debugging tools to troubleshoot and optimize your code

✓
1
🐛 Enable Debugging Mode

Configure the debugger for your Blue Pill project:

  • Click Run → Debug Configurations
  • Select appropriate debugger (ST-Link GDB Server)
  • Verify connection settings
  • Click Debug to start debugging session
✓
2
🔍 Use Debugging Tools

Utilize breakpoints and variable monitoring for effective debugging:

  • Set breakpoints by clicking in the left margin of code editor
  • Use Variables tab to monitor real-time values
  • Check Registers tab for peripheral registers
  • Use step-through debugging (F5/F6) to trace execution
🎯 Debug Features

Live expressions, memory browser, and real-time variable modification available.
